1/13 Git Flow

Инструкция

Кликате по кнопке и увидите git flow А команды будут выполняться в окне внизу

Слидите внимательно:

  • В консоле будет отображена команда и её выполнение.
  • В окнах будет отображен процесс гит с текущими и последующими шагами

Что же такое git и его рабочий процесс

Разберем на примере файла:

Time Status
July 9, 10:00 Create file tasks.txt.
July 9, 10:05 Add paragraph to file tasks.txt.
July 9, 10:10 Format the content of tasks.txt.
… …

Git позволяет нам фиксировать каждое изменение в файле: построчно и по времени

Git основы

Git инициализируется в рабочей директории командой git init. После инициализации, git начинает контролировать директорию

В git присутсвует 3 основных понятия: рабочая директория, область перемещения, удаленный репозиторий.

Кликаем по кнопке и посмотрим на процессы.

git init далее

2/13 Первые изменения

Создаем файлы в директории (в примере мы используем touch , но вы можете разворачивать наши базовые проекты)

Для примера создадим файл tasks.txt. с тем, что нужно сделать по кода (to-do список)

Git следит за рабочей папкой и уже заметил выполнение нашей команды

touch tasks.txt далее

3/13 Добавление изменений в область перемещения

Зафиксировать изменения и переместить на удаленный репозиторий можно сделав commit, то есть соверщить процесс передачи данных на удаленный репозиторий. Для этого мы должны сделать несколько шагов. 1 - добавить изменения из директории в область изменений. staging area.

Для добавления в область изменений используйте команду git add. и укажите файл tasks.txt или все файлы используя .

git add tasks.txt далее

4/13 Создание commit

2 - запаковка изменений для отправки в удаленный репозиторий.

Для этого используем комманду git commit.

Для простого поиска ваших изменений добавьте к git commit специальное сообщение. Используйте опцию –m.

Следующая команда создает коммит к добавленному файлу Add file tasks.txt.

git commit –m "Add file tasks.txt" git push origin master" далее

5/13 Изменение файла

Дополним файл 1 строкой используя echo >>

Добавим задание Посчитать случайные интервалы. в конец файла tasks.txt используя echo command.

echo "Посчитать случайные интервалы" >> tasks.txt далее

6/13 Git status

Очень полезная команда git status. Она показывает изменения и ошибка (подскажет как их победить)

Посмотрим результат команды в консоле. git status покажет нам, что файл был изменен. И подскажет команды для решения проблемы tasks.txt имеет статус модифицированной команды, так как мы добавили туда строку

git status далее

7/13 Список изменений

Команда git status только показывает изменения. Если мы захотим узнать наши изменения, то воспользуемся команандой git diff.

Команда git diff покажет, зеленым - строчки, которые мы добавили, красным - строки, которые мы удалили.

git diff далее

8/13 Добавим

Нам нужно дополнить нашими изменениями в tasks.txt в удаленном репозитории. Начнем с подготовки области перемещения git add.

git add tasks.txt далее

9/13 Git status 2

Используем команду git status чтобы посмотреть изменения. Файл tasks.txt ещё в статусе изменения, но файл уже зеленый, что показывает его добавление в область перемещения

git status далее

10/13 Изменения 2

Мы уже смотрели изменения командой git diff и получали результаты о наших модификациях.

Мы также можем посмотреть изменения в области перемещения, используем опцию ––cached.

git diff ––cached далее

11/13 Ещё commit

Добавим изменения на наш удаленный репозиторий Запакуем наши изменения git commit и отправим на удаленный репозиторий

git commit –m "Add first task" git push origin master" Next

12/13 Обзор commits

git log команда показывает все коммиты, которые были сделаны

git log это одна из самых разнообразных команды, имеет много ключей для использования. Посмотрим ключ ––oneline для обзова в коротком формате

git log ––oneline далее
